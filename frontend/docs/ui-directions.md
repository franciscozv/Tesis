# UI Direction - Sistema de Gestión Iglesia

## Objetivo
Diseñar una interfaz web institucional, sobria, cálida, moderna y accesible para una aplicación de gestión de iglesia.

## Contexto del proyecto
La aplicación está orientada a la gestión administrativa y operativa de una iglesia.
Debe transmitir:
- confianza
- orden
- claridad
- cercanía
- respeto
- profesionalismo

La interfaz debe ser fácil de usar para personas de distintas edades y niveles de experiencia digital.

## Identidad visual deseada
Quiero una interfaz:
- institucional
- elegante
- sobria
- limpia
- moderna
- accesible
- distinta al típico dashboard SaaS genérico

## Dirección estética
La UI debe inspirarse en la sobriedad y claridad de Radix UI:
- componentes limpios
- estructura consistente
- buena jerarquía visual
- accesibilidad
- neutralidad elegante

Pero no debe verse fría ni como herramienta para desarrolladores.
Debe adaptarse a una aplicación de iglesia con una identidad más cálida y confiable.

## Reglas importantes
- No usar morado como color principal
- No usar gradientes exagerados
- No usar glassmorphism
- No usar estilo fintech
- No usar apariencia de startup SaaS genérica
- No sobrecargar la interfaz con animaciones innecesarias
- No abusar de sombras fuertes

## Logo e imagen de referencia
- Analiza el logo/icono proporcionado por el usuario
- Usa el logo como inspiración para definir la paleta visual
- No copies el logo en exceso dentro de la interfaz
- Usa también cualquier imagen de referencia que esté en /context para entender el estilo buscado

## Paleta de color
Debes elegir una paleta basada en el logo, definiendo:
- color primario
- color secundario
- color de acento
- color de fondo
- color de superficies/cards
- color de bordes
- color de texto principal
- color de texto secundario
- color de estados (éxito, advertencia, error)

Prioriza colores institucionales, sobrios y accesibles.

## Componentes y herramientas
Al trabajar en la UI:
- usa shadcn/ui como base
- aprovecha el MCP de shadcn para buscar componentes y bloques adecuados
- mantén consistencia en spacing, bordes, tamaños y tipografía
- reutiliza componentes
- evita estilos improvisados o incoherentes

## Testing y validación
Después de proponer o aplicar cambios:
- usa Playwright para revisar la pantalla en vivo
- prueba desktop
- prueba tablet
- prueba mobile
- detecta problemas de layout, overlap, scroll horizontal y accesibilidad
- revisa estados hover, focus, disabled y errores de formulario

## Pantallas prioritarias
Aplicar esta dirección visual especialmente en:
- login
- dashboard
- sidebar
- listado de miembros
- formularios
- actividades
- reportes

## Qué espero al pedir un rediseño
Cuando te pida mejorar una pantalla:
1. analiza el logo y las referencias visuales
2. define una dirección visual coherente
3. elige colores basados en el logo
4. usa componentes apropiados de shadcn
5. mantén estilo institucional inspirado en Radix UI
6. evita look genérico SaaS
7. prueba el resultado con Playwright
8. entrega observaciones de diseño si detectas problemas

## Resultado esperado
La interfaz debe sentirse:
- confiable
- cálida
- clara
- elegante
- seria
- moderna
- usable
- coherente visualmente
